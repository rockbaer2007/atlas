# ATLAS Framework

# Architecture Decision Records (ADR)

This directory contains the Architecture Decision Records (ADRs) of the
ATLAS Framework.

Each ADR documents a significant architectural decision together with its
context, rationale and consequences.

---

# Purpose

Architecture Decision Records explain **why** architectural decisions were
made.

They are intended to provide long-term documentation for contributors and
maintainers.

Every ADR should answer the following questions:

- What problem existed?
- Which alternatives were considered?
- Which solution was chosen?
- Why was this solution selected?
- What are the consequences?

---

# ADR Status

Possible ADR states are:

| Status | Meaning |
|---------|---------|
| Proposed | Decision under discussion |
| Accepted | Official project decision |
| Superseded | Replaced by a newer ADR |
| Deprecated | No longer recommended |

# Writing Guidelines

Each ADR should include:

1. Context
2. Decision
3. Consequences
4. References (optional)

The document should focus on architectural decisions rather than
implementation details.
